audio-editor Module

Defined in: app/assets/javascripts/audio_editor.js:1

This module contains javascript functions used to animate the Audio Editor. The editing of an audio consists in cutting and connecting together different elements of type audio (referred to as audio components): each component has a unique identifier, to be distinguished by the ID in the database of the corresponding audio.

Each audio component contains a cutter (used to select a portion of the original audio), the standard player controls, and a sorting handle. To use the controls of a component, first it must be selected (using the function selectAudioEditorComponent): if a component is not selected, it's shown with opacity and without the player controls. The functionality of sorting components is handled in audioEditorDocumentReadyGeneral; the functionalities of player commands, together with the most important functionalities of the cutter, are located in the module players (class PlayersAudioEditor); the only functionalities of the cutter located in this module are found in the class AudioEditorCutters and in the method audioEditorDocumentReadyCutters (functionality of precision arrows).

On the top of the right column are positioned the time durations (updated with changeDurationAudioEditorComponent), ad the button 'plus' used to open the audio gallery (see the class AudioEditorGalleries and the method audioEditorDocumentReadyGeneral).

On the bottom of the right column are positioned the controls for the global preview (which plays all the components globally). Similarly to the module video-editor, before playing the global preview the system must enter in preview mode: the difference is that in the Audio Editor this mode is switched automatically by the system, whereas in the Video Editor it's the user who decides to enter and leave the preview mode. To enter in preview mode we use the method enterAudioEditorPreviewMode, which calls switchAudioComponentsToPreviewMode or each component. A component in preview mode can be selected (using a differente method respect to the normal component selection: see selectAudioEditorComponentInPreviewMode). Each component in the global preview is played using the method startAudioEditorPreview (to which we pass the component to start with); the main part of the functionality of passing from a component to the next one during the global preview is handled in the module players (more specificly, in the method initializeActionOfMediaTimeUpdaterInAudioEditor.

As for the other Element Editors (image-editor, video-editor) the core of the process of committing changes is handled in the module media-element-editor (more specificly in the class MediaElementEditorForms); the part of this functionality specific for the Audio Editor is handled in audioEditorDocumentReadyCommit.
