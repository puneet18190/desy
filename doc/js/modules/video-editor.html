video-editor Module

Defined in: app/assets/javascripts/video_editor.js:1

The Video Editor is structured as follows: centered in the middle of the Editor is located the preview screen, below it the components timeline and the audio track, and on the right the preview column which contains also global statistics about the video.

A video created with the Video Editor can be composed by three types of components (and optionally an audio track):

  • a video component is an element of type video extracted from the user's gallery, associated to an initial and final point
  • a image component is an element of type image extracted from the user's gallery, associated to a duration in seconds (the image is held in the video for a number of seconds equal to the component's duration); the image is centered and cropped maintaining its original proportions, to make it fit in the video screen (which has proportions 16/9)
  • a text component is a centered title for which the user chooses a background color, a font color and a duration (which has the same interpretation as for image components).

The resulting video will be the concatenation of all the components inside the timeline, with optionally the chosen audio track as background audio. On the timeline the user may perform the following actions:

  • add a new component (see class VideoEditorAddComponents) or replace an existing one, even without maintaining its original type (see class VideoEditorReplaceComponents): these functionalities are initialized in videoEditorDocumentReadyAddComponent
  • remove a component from the timeline (initialized in videoEditorDocumentReadyRemoveComponent)
  • sort and change the order of the components (initialized in videoEditorDocumentReadyInitialization)
  • cut a video component (change its initial and final point) or change duration of an image or text compoent (both these functionalities are initialized in videoEditorDocumentReadyCutters and implemented in the class VideoEditorCutters).

Each component is provided of its own identifier (the same used in audio-editor), that is unique and doesn't change on any operation performed by the user. Moreover, regardless of its type, a component is strictly linked with two external accessories:

  • a cutter (whose HTML id is video component [identifier] cutter): this item is normally hidden, when requested it appears below the timeline and is used to cut a video component or change the duration of an image or text component (class VideoEditorCutters)
  • a preview clip (whose HTML id is video component [identifier] preview): this item is hidden inside the preview screen, and it's used
    • to provide the user of a big clip of the component while handling it (functionality initialized in videoEditorDocumentReadyComponentsCommon)
    • to play a video component while cutting it (initialized in the method playersDocumentReadyVideoEditor in the module players)
    • to be shown in the global preview (see class VideoEditorPreview).

The method that extracts the identifier from a component is getVideoComponentIdentifier (it works receiving as parameter either the component or its cutter or preview clip).

While the user is working, the preview clip visible in the preview screen corresponds to the last component selected by the user. A component gets selected either if the user keeps the mouse on it for more than half a second (using the method startVideoEditorPreviewClipWithDelay, which avoids compulsive changes inside the preview screen when the user passes with the mouse over the timeline), or immediately on the actions of sorting and cutting (using the method startVideoEditorPreviewClip): both behaviors are initialized in videoEditorDocumentReadyComponentsCommon. To the preview clip of a video component is also associated a method (loadVideoComponentIfNotLoadedYet) that loads the HTML5 video tag only when necessary: this, similarly to audio-editor, avoids overloading of the HTML.

The component gallery used inside the Video Editor (initialized in initializeMixedGalleryInVideoEditor) is the only gallery in the application which contains mixed types of elements. It's divided into three sections, one for each kind of component: the sections for video and image components have the same external behavior of normal image and video galleries (see the module galleries), whereas the section for text components is a peculiar text editor (see the class VideoEditorTextComponentEditor). The component gallery (together with the regular audio gallery for the audio track) is initialized in videoEditorDocumentReadyGalleries, and its functionality defined in the methods of VideoEditorGalleries (for instance, the method to switch from a section to another).

The method videoEditorDocumentReadyAddComponent initializes the general procedure to add or replace a component. The system sets a HTML data that records if the component gallery was opened to replace or add a component: depending on this data, when the user picks a component from the gallery it's called the corresponding method in VideoEditorAddComponents or in VideoEditorReplaceComponents.



As for the other Element Editors (image-editor, audio-editor) the core of the process of committing changes is handled in the module media-element-editor (more specificly in the class MediaElementEditorForms); the part of this functionality specific for the Video Editor is handled in videoEditorDocumentReadyCommit.
